# Prompt Patterns

Use these compact patterns when improving rough image requests.

## Prototype Or UI Mockup

Create a high-fidelity [platform/device] UI prototype screenshot for [product/domain]. Show [main screen/state] with [key components]. Use realistic Chinese interface copy, clear information hierarchy, modern product design, consistent spacing, crisp typography, and production-ready visual polish. Color/style: [style]. Aspect ratio: [ratio]. No placeholder gibberish.

## Requirement-Only Prototype

Create a high-fidelity [platform/device] product prototype screenshot based on this requirement: [requirement summary]. Screen: [chosen screen/state]. User goal: [goal]. Layout: [navigation + main regions]. Components: [component list]. Use realistic Chinese UI copy, clear hierarchy, production-grade spacing, consistent design system, and polished modern interface details. Style: [style]. No placeholder gibberish.

## PRD-To-New-Prototype

Create a high-fidelity [platform/device] prototype for [product/domain] based on this PRD summary: [brief]. Target user: [user]. Core workflow: [workflow]. Render the [screen/state] screen. Information architecture: [navigation], [primary module], [secondary modules], [status/feedback surfaces], [primary CTA]. Include realistic Chinese UI copy for [important labels/data/actions]. Visual direction: [selected direction]. Interaction cues: [state transitions/microinteractions]. Use credible product density, precise spacing, readable typography, coherent icons, and no placeholder gibberish.

## Creative Direction Exploration

Generate a visually ambitious [platform/device] UI concept for [product/domain]. Product goal: [goal]. Required functions: [functions]. Creative intensity: [level]. Direction: [premium editorial / immersive spatial / motion-first / domain-native / experimental]. Composition: [layout idea]. Interaction concept: [gesture/state/transition]. Materials and atmosphere: [materials/color/lighting]. Keep the interface usable and grounded in the PRD, with realistic Chinese copy and clear hierarchy. Avoid generic purple gradients, random neon, meaningless cards, and placeholder gibberish.

## Motion-First Interaction Still

Create a still-image prototype that communicates an interaction for [product/domain]. Show [main screen] with visible motion cues: [before-after state], [transition ghosting], [gesture handle], [progress path], and [feedback microcopy]. The design should feel [style/intensity] while remaining understandable as a product UI. Include realistic Chinese copy, crisp component boundaries, and a clear action-result relationship.

## Existing Prototype Iteration

Using the provided prototype image as the visual reference, create an updated version that preserves the original screen structure, design language, color palette, typography feel, spacing rhythm, navigation, and unchanged modules. Modify only [requested feature/module/change]. Add/update the UI copy as follows: [copy]. If the user did not request a new visual style, keep the original style. Maintain realistic Chinese interface text and a coherent product prototype screenshot.

## HTML Prototype Slice Edit

Using this PNG slice rendered from an HTML prototype as the visual reference, preserve the original HTML prototype style, layout density, typography feel, component hierarchy, spacing rhythm, colors, and unchanged modules. Slice source: [manifest index / selector / visible text]. Modify only [requested region or feature]. If a mask is provided, edit only the masked area and preserve every unmasked region exactly. Keep the result looking like the same rendered prototype screen, with realistic Chinese interface text and no unrelated changes.

## Faithful Local Edit

Edit only the masked area of the provided image. Preserve every unmasked region exactly: layout, colors, typography feel, spacing, screenshot proportions, icons, text, cards, and background. In the masked area, [requested change]. Match the original UI/product style and make the new element look native to the existing design. Do not change any unrelated content.

## Semantic Local Edit

Edit only the masked area identified as [semantic region description] in the provided image. The target region corresponds to [visual landmarks / coordinates]. Preserve every unmasked region exactly, including layout, colors, typography feel, spacing, screenshot proportions, icons, text, cards, and background. Apply this change inside the target region: [requested change]. Match the original UI/product style and make the new element look native to the existing design.

## Multi-Region Local Edit

Edit only the masked regions in the provided image. Region 1: [semantic region / coordinates] -> [change]. Region 2: [semantic region / coordinates] -> [change]. Region 3: [semantic region / coordinates] -> [change]. Preserve every unmasked region exactly, including all unchanged text, layout, colors, typography feel, spacing, icons, cards, and background. Make all new elements match the original UI/product style and look like native parts of the same screen.

## Free Redesign

Create a new high-fidelity design based on [goal/reference]. You may reinterpret the layout and visual style according to [new direction]. Preserve only the product intent and key functional requirements: [requirements]. Produce a polished, coherent new page/design with realistic Chinese copy, strong hierarchy, and production-grade visual details.

## Design Poster

Create a [orientation] Chinese poster for [theme/product/event]. Main visual: [subject]. Composition: [layout]. Include exact text: [text]. Style: [visual style]. Use strong hierarchy, legible Chinese typography, balanced negative space, and polished commercial art direction.

## Reference Edit

Edit the provided reference image while preserving [must-keep elements]. Change [target change]. Maintain realistic lighting, perspective, edges, materials, and visual consistency. Do not alter [protected details].
